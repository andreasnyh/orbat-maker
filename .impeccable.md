# Impeccable Design Context

## Design Context

### Users
Milsim (military simulation) players and unit organizers who need to build, manage, and share Orders of Battle. They work with personnel rosters, organizational templates, and role assignments. Users are typically detail-oriented gamers comfortable with structured data and military terminology. They use this tool before operations to plan unit composition, often sharing results via Discord or TeamSpeak.

### Brand Personality
**Tactical, precise, functional.** The interface should feel like a well-made tool — purposeful, no-nonsense, and reliable. Every element earns its place. Military undertones are present but understated; this is a planning tool, not a game.

### Emotional Goals
**Confidence and control.** Users should feel in command of their data. The interface responds predictably, information is where they expect it, and actions feel deliberate. No ambiguity, no surprises.

### Aesthetic Direction
- **Visual tone**: Utilitarian with polish. Dense but not cluttered. Information-forward.
- **Themes**: Three established themes — Midnight (dark blue), Olive (military green), Sand (warm light). All use semantic CSS custom properties.
- **Typography**: Barlow Condensed for display/headings (military stencil energy), JetBrains Mono for data (precision, readability).
- **Color strategy**: Muted palettes with a single accent color per theme. Equipment highlighted in amber/brown.
- **Background texture**: Subtle topographic map pattern at low opacity — adds depth without distraction.
- **Anti-references**: Avoid playful/rounded SaaS aesthetics, gamified UI, excessive animations, or anything that feels toy-like.

### Design Principles

1. **Function dictates form.** Every visual choice serves a purpose — aiding scanning, grouping, or interaction. Decoration that doesn't improve usability gets cut.

2. **Information density over whitespace.** Users manage many units, people, and slots. Respect screen real estate. Compact layouts with clear hierarchy beat spacious ones with less data visible.

3. **Predictable and deliberate.** Interactions should feel direct and mechanical. Minimal animation, instant feedback. Drag-and-drop is the core interaction — everything else stays out of its way.

4. **Theme-native, not theme-agnostic.** All three themes are first-class. Never hardcode colors. Use semantic tokens so every theme feels intentional, not adapted.

5. **Accessible by default.** Target WCAG AA. Good contrast ratios, keyboard navigation, focus indicators, and reduced-motion support. Military precision applies to inclusion too.

## Technical Notes

- **Token system**: CSS custom properties scoped by `[data-theme]`, registered in Tailwind 4 via `@theme inline`
- **Semantic classes**: `.card` for panels, `text-body`/`text-dim`/`text-accent` for text hierarchy, `bg-page`/`bg-panel` for surfaces
- **Opacity modifiers**: Supported on all tokens (`bg-accent/50`, `border-trim/75`)
- **Animation policy**: Minimal. `fade-in`/`fade-out` only. Respects `prefers-reduced-motion`.
- **Font loading**: Google Fonts with `display=swap`, preconnected
